### Sequence Diagram

1. 키오스크: 투표 목록 전체 불러오기

``` sequence
Front Server ->> BackEnd Server: 투표 목록 데이터 요청
BackEnd Server ->> DataBase: 데이터 요청
DataBase ->> BackEnd Server: 데이터 응답
BackEnd Server -->> Front Server: 투표 목록 데이터 응답
Front Server ->> Client(Kiosk): 랜더링
```

2. 키오스크: 투표하기

```sequence
Client(Kiosk) ->> BackEnd Server: 투표하기
BackEnd Server ->> DataBase: 데이터 전달
DataBase ->> BackEnd Server: 데이터 응답
BackEnd Server -->> Client(Kiosk): 투표하기 성공
```



3. 웹: 재고 목록 전체 불러오기

```sequence
Front Server ->> BackEnd Server: 재고 데이터 요청
BackEnd Server ->> DataBase: 데이터 요청
DataBase ->> BackEnd Server: 데이터 응답
BackEnd Server -->> Front Server: 재고 데이터 응답
Front Server ->> Client(Web): 랜더링
```



4. 웹: 회원가입 및 로그인

```sequence
Client(Web) ->> BackEnd Server: 가입 정보 전송
BackEnd Server ->> DataBase: 데이터 요청, 유효성 검사
DataBase ->> BackEnd Server: 데이터 응답
BackEnd Server -->> Client(Web): 회원가입 성공
Client(Web) ->> BackEnd Server: 로그인 정보 전송
BackEnd Server ->> DataBase: 데이터 요청, 유효성 검사
DataBase ->> BackEnd Server: 데이터 응답
BackEnd Server -->> Client(Web): 로그인 토큰
```

4. 웹: 투표 생성

```sequence
Client(Web) ->> BackEnd Server: 투표 생성 정보 전송
BackEnd Server ->> DataBase: 데이터 요청, 유효성 검사
DataBase ->> BackEnd Server: 데이터 응답
BackEnd Server -->> Client(Web): 투표 생성 성공
```
